1. DIAGRAMME USE CASE (Cas d'utilisation)
==========================================

**Acteurs principaux :**
- **Agent Immobilier** : Professionnel qui évalue des propriétés pour ses clients
- **Investisseur** : Personne cherchant des opportunités d'investissement rentables
- **Vendeur Particulier** : Propriétaire voulant estimer son bien pour le vendre
- **Acheteur Particulier** : Personne cherchant à acheter et vérifiant la pertinence des prix
- **Administrateur Système** : Gère la maintenance et les mises à jour

**Cas d'utilisation principaux :**

**UC1 - Prédire le Prix d'un Bien**
- Acteur : Agent Immobilier, Investisseur, Vendeur, Acheteur
- Description : Estimer le prix de vente d'un bien immobilier
- Préconditions : Utilisateur authentifié, données du bien disponibles
- Postconditions : Prix prédit généré avec intervalle de confiance

**UC2 - Analyser le Marché Local**
- Acteur : Investisseur, Agent Immobilier
- Description : Obtenir des statistiques et tendances sur une zone géographique
- Préconditions : Localisation définie
- Postconditions : Rapport marché généré

**UC3 - Comparer des Biens Similaires**
- Acteur : Acheteur, Investisseur, Agent Immobilier
- Description : Trouver et comparer des propriétés similaires dans la même zone
- Préconditions : Caractéristiques du bien de référence
- Postconditions : Liste de biens comparables avec prix

**UC4 - Sauvegarder une Évaluation**
- Acteur : Agent Immobilier, Investisseur
- Description : Enregistrer une prédiction pour référence future
- Préconditions : Prédiction effectuée, utilisateur connecté
- Postconditions : Évaluation sauvegardée dans le profil

**UC5 - Exporter un Rapport d'Évaluation**
- Acteur : Agent Immobilier, Investisseur
- Description : Générer un PDF détaillé de l'évaluation
- Préconditions : Évaluation complète disponible
- Postconditions : Rapport PDF généré et téléchargé

**UC6 - Gérer son Profil Utilisateur**
- Acteur : Tous les utilisateurs
- Description : Mettre à jour informations personnelles et préférences
- Préconditions : Utilisateur authentifié
- Postconditions : Profil utilisateur mis à jour

**UC7 - Maintenir le Modèle ML**
- Acteur : Administrateur Système
- Description : Mettre à jour et réentraîner le modèle de prédiction
- Préconditions : Nouvelles données disponibles
- Postconditions : Modèle mis à jour en production

**Cas d'utilisation secondaires :**

**UC8 - Authentification**
- Acteur : Tous les utilisateurs
- Description : Se connecter au système

**UC9 - Géolocaliser une Adresse**
- Acteur : Tous les utilisateurs
- Description : Convertir une adresse en coordonnées GPS

**UC10 - Valider les Données d'Entrée**
- Acteur : Système
- Description : Vérifier la cohérence des caractéristiques saisies

**Relations entre cas d'utilisation :**
- <<include>> : UC8 inclus dans tous les UC principaux
- <<extend>> : UC10 étend UC1, UC2, UC3
- <<generalization>> : UC1 généralise UC2, UC3

3. DIAGRAMME DE SÉQUENCE
=========================

**Séquence 1 : Prédiction de Prix par un Agent Immobilier**

**Acteurs :**
- Agent (AgentImmobilier)
- InterfaceWeb (Frontend)
- API (ServicePrediction)
- ModeleML (ModelePrediction)
- BDD (Base de données)

**Scénario nominal :**
```
1. Agent → InterfaceWeb : saisitCaracteristiquesBien()
2. InterfaceWeb → API : POST /predict (donneesBien)
3. API → API : validerDonnees(donneesBien)
4. API → BDD : getQuartierInfo(adresse)
5. BDD → API : quartierInfo
6. API → API : pretraitementDonnees(donneesBien, quartierInfo)
7. API → ModeleML : predire(featuresPretraitees)
8. ModeleML → ModeleML : calculerPrediction()
9. ModeleML → API : prediction(prix, confiance)
10. API → API : formaterResultats(prediction)
11. API → InterfaceWeb : response(prediction)
12. InterfaceWeb → Agent : afficherPrediction(prixEstime, intervalleConfiance)
```

**Scénarios d'exception :**
- **Données invalides :** API → InterfaceWeb : erreur("Données incomplètes")
- **Modèle indisponible :** API → InterfaceWeb : erreur("Service temporairement indisponible")
- **Timeout :** API → InterfaceWeb : erreur("Délai d'attente dépassé")

**Séquence 2 : Analyse de Marché par un Investisseur**

**Acteurs :**
- Investisseur
- InterfaceMobile
- ServiceMarche
- APIExterne
- CacheRedis

**Scénario :**
```
1. Investisseur → InterfaceMobile : demandeAnalyseQuartier(quartier)
2. InterfaceMobile → ServiceMarche : getStatistiquesQuartier(quartier)
3. ServiceMarche → CacheRedis : get(quartier_stats)
4. CacheRedis → ServiceMarche : null (données non en cache)
5. ServiceMarche → BDD : getBiensQuartier(quartier)
6. BDD → ServiceMarche : listeBiens
7. ServiceMarche → APIExterne : getIndicesPrix(quartier)
8. APIExterne → ServiceMarche : indicesMarche
9. ServiceMarche → ServiceMarche : calculerStatistiques(listeBiens, indicesMarche)
10. ServiceMarche → CacheRedis : set(quartier_stats, statistiques, TTL=1h)
11. ServiceMarche → InterfaceMobile : statistiquesQuartier
12. InterfaceMobile → Investisseur : afficherAnalyse(graphiques, tendances)
```

**Séquence 3 : Génération de Rapport d'Évaluation**

**Acteurs :**
- Agent
- ServiceEvaluation
- ServicePrediction
- GenerateurPDF
- StockageS3

**Scénario :**
```
1. Agent → ServiceEvaluation : demanderRapport(evaluationId)
2. ServiceEvaluation → BDD : getEvaluation(evaluationId)
3. BDD → ServiceEvaluation : evaluation
4. ServiceEvaluation → ServicePrediction : getDetailsPrediction(evaluation.bien)
5. ServicePrediction → ServiceEvaluation : detailsPrediction
6. ServiceEvaluation → ServiceMarche : getBiensSimilaires(evaluation.bien)
7. ServiceMarche → ServiceEvaluation : biensSimilaires
8. ServiceEvaluation → GenerateurPDF : genererRapport(evaluation, details, similaires)
9. GenerateurPDF → GenerateurPDF : creerLayout()
10. GenerateurPDF → GenerateurPDF : ajouterSections()
11. GenerateurPDF → GenerateurPDF : convertirPDF()
12. GenerateurPDF → ServiceEvaluation : fichierPDF
13. ServiceEvaluation → StockageS3 : upload(fichierPDF, rapportId)
14. StockageS3 → ServiceEvaluation : urlPDF
15. ServiceEvaluation → Agent : urlRapportPDF
```

**Séquence 4 : Mise à Jour du Modèle ML**

**Acteurs :**
- Admin
- ServiceML
- DataSource
- ModeleML
- Monitoring

**Scénario :**
```
1. Admin → ServiceML : lancerMiseAJourModele()
2. ServiceML → DataSource : getNouvellesDonnees(dateDerniereMiseAJour)
3. DataSource → ServiceML : nouvellesDonnees
4. ServiceML → ServiceML : nettoyerDonnees(nouvellesDonnees)
5. ServiceML → ServiceML : diviserDataset(dataset, train=0.8, test=0.2)
6. ServiceML → ModeleML : entrainer(trainData)
7. ModeleML → ModeleML : optimiserHyperparametres()
8. ModeleML → ServiceML : modeleEntraîne
9. ServiceML → ModeleML : valider(testData)
10. ModeleML → ServiceML : metriquesValidation
11. ServiceML → Monitoring : comparerPerformances(metriquesValidation)
12. Monitoring → ServiceML : amelorationSignificative(true)
13. ServiceML → ServiceML : deployerModele(modeleEntraîne)
14. ServiceML → Admin : confirmationMiseAJour()
```

**Séquence 5 : Authentification et Session Utilisateur**

**Acteurs :**
- Utilisateur
- AuthService
- JWTService
- BDD

**Scénario :**
```
1. Utilisateur → AuthService : login(email, password)
2. AuthService → BDD : findUserByEmail(email)
3. BDD → AuthService : user
4. AuthService → AuthService : verifyPassword(password, user.passwordHash)
5. AuthService → JWTService : generateToken(user)
6. JWTService → AuthService : jwtToken
7. AuthService → AuthService : createSession(user, jwtToken)
8. AuthService → BDD : saveSession(session)
9. AuthService → Utilisateur : authResponse(jwtToken, userInfo)
10. Utilisateur → AuthService : requeteProtegee(jwtToken)
11. AuthService → JWTService : validateToken(jwtToken)
12. JWTService → AuthService : tokenValid
13. AuthService → BDD : getSession(token)
14. BDD → AuthService : session
15. AuthService → Utilisateur : responseDonnees()
```

**Points de synchronisation :**
- **[concurrent]** : Plusieurs utilisateurs peuvent faire des prédictions simultanément
- **[critical]** : Mise à jour du modèle (pas de prédictions pendant déploiement)
- **[atomic]** : Sauvegarde d'une évaluation (transaction complète)

**Messages asynchrones :**
- Notification de changement de prix
- Rapport de performance quotidien
- Alertes de maintenance système
